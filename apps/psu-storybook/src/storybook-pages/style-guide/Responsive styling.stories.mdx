<h1>Responsive design using sx and extraSx</h1>
<br />
<br />
<br />

<h3>
  Like Styled System, we use responsive array syntax.{' '}
  <a href="https://theme-ui.com/sx-prop#responsive-values">
    Click here for docs
  </a>
</h3>

<p>
  Responsiveness can be achieved with a shorthand syntax for writing
  mobile-first responsive styles using arrays as values.
</p>

```jsx
<Flex extraSx={{gridColumn: ['1/5', '2/8', '3/11', '3/11']}}>
{...}
</Flex>
```

<p>
  In this example we're setting our gridColumn to 4 different values depending
  on the breakpoint. This is how it reads :
</p>

```jsx
gridColumn: [
  'value at sm breakpoint',
  'value at md breakpoint',
  'value at lg breakpoint',
  'value at xl breakpoint',
];
```

<p>
Responsive array syntax can be used for any css property inside the sx prop or the extraSx prop.

</p>

```jsx
<Flex extraSx={{ p: [1, 2, 3, 4], m: [1, 2, 3, 4] }}>
  <div sx={{ backgroundColor: ['link', 'nittanyNavy', 'coalyGray', 'white'] }}>
    <h1
      sx={{
        lineHeight: ['120', '140', '150', '150'],
        fontSize: [32, 36, 40, 44],
      }}
    >
      Heading
    </h1>
  </div>
</Flex>
```

<h3>Media Queries</h3>
<p>
  If you prefer standard CSS media query syntax, you can use nested objects for
  responsive styles as well.
</p>

```jsx
<div
  sx={{
    width: '100%',
    '@media screen and (min-width: 40em)': {
      width: '50%',
    },
  }}
/>
```
