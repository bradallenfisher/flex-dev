<h2 style={{ border: 'none', marginTop: '50px' }}>
  Accessing the breakpoint tokens
</h2>

<h4>All breakpoints can be accessed using array syntax inside sx or extraSx</h4>

<p>
  **Access any breakpoint token by using the sx or extraSx prop. These can then
  be used in combination with responsive array values and variants. For
  instance, you can create reusable typographic styles that don't pollute the
  global CSS scope. Responsive array values work with other CSS properties
  inside the sx prop as well.
</p>

```jsx
//for functional components
<Flex extraSx={{ backgroundColor: ['link', 'nittanyNavy', 'link', 'white'] }}>...</Flex>

{/* in the above, backgroundColor is:
'link' at sm breakpoint,
'nittanyNavy' at md breakpoint,
'link' at lg breakpoint,
and 'white' at xl breakpoint */}


//for html elements
<div sx={{fontSize: [4, 7, "13px", "3rem"]}}/>
<div sx={{width: ["100%", "75%", "400px", "10vw"]}}/>
...
```

<p>
  In the above example, inside the sx prop - each css property value in the
  property array will correspond to the breakpoint at that index in the
  breakpoint token array.
</p>

<p>
  To be more specific, in the first div - the fontSize will be set to..
  <ul>
    <li>4 at the sm breakpoint</li>
    <li>7 at the md breakpoint</li>
    <li>13px at the large breakpoint</li>
    <li>3rem at the xl breakpoint</li>
  </ul>
</p>
<p>These CSS properties will use values defined in the theme, when available just like using CSS properties inside the sx prop withouth the breakpoint array.</p>
</div>
